6.2. Common Procedures

  • How to Restart a Service:
    • Via Services Management Console (GUI):
      1. Press Win + R, type services.msc, and press Enter.
      2. Locate the desired service (e.g., Apache2.4, MySQL).
      3. Right-click the service and select Restart.
    • Via Command Prompt (CMD) / PowerShell (Run as Administrator): 
      net stop "[Service Name]"
        net start "[Service Name]"

      Example for Apache: net stop "Apache2.4" then net start "Apache2.4"

      Example for MySQL: net stop "MySQL" then net start "MySQL"

  • How to Check Disk Space: 
    wmic logicaldisk get caption,size,freespace

    Alternatively, use File Explorer: Open "This PC" to see drive usage.

  • How to Check Running Processes:
    • Task Manager (GUI): Press Ctrl + Shift + Esc. Go to the "Processes" or "Details" tab.
    • Command Prompt (CMD) / PowerShell: 
      tasklist

      To find specific processes or sort (PowerShell for top 10 by CPU): 

      Get-Process | Sort-Object -Property CPU -Descending | Select-Object -First 10

  • How to Check Log Files:
    • Main System Logs (Event Viewer):
      1. Press Win + R, type eventvwr.msc, and press Enter.
      2. Navigate to Windows Logs (e.g., Application, System) or Applications and Services Logs for specific service events.
    • XAMPP/Application Logs:
      • Apache Error Log: C:\xampp\apache\logs\error.log
      • Apache Access Log: C:\xampp\apache\logs\access.log
      • MySQL Error Log: C:\xampp\mysql\data\[YourComputerName].err
      • PHP Error Log: Check php.ini for error_log path, often in C:\xampp\php\logs\ or directly in C:\xampp\apache\logs\.
  • How to Perform a Graceful Reboot: 
    shutdown /r /t 0

    This command initiates an immediate restart. For a scheduled restart, change 0 to a number of seconds (e.g., shutdown /r /t 30 for 30 seconds).

  • Adding/Removing User Accounts:
    • Via Control Panel: Control Panel > User Accounts > Manage another account.
    • Via Computer Management: Win + X > Computer Management > Local Users and Groups.
    • Via Command Prompt (CMD) / PowerShell (Run as Administrator): 
      net user [username] [password] /add
      net localgroup Administrators [username] /add
  • Troubleshooting Common Issues:
    • High CPU Usage:
      • Open Task Manager (Ctrl + Shift + Esc). Go to the "Processes" or "Details" tab and sort by CPU to identify culprits.
      • Check application-specific logs for processes consuming high CPU.
    • Disk Space Full:
      • Use File Explorer to visually identify large folders/files.
      • Use wmic logicaldisk get caption,size,freespace in CMD.
      • Review application logs (especially Apache/MySQL) for excessive log file growth.
      • Consider a disk cleanup tool like Windows' built-in Disk Cleanup or third-party tools like WinDirStat.
    • Service Not Starting:
      • Check Services Management Console (services.msc) for the service status and startup type. Try to start it manually.
      • Review the Event Viewer (eventvwr.msc), particularly Windows Logs > Application and System, for error messages related to the service.
      • Check service-specific logs (e.g., Apache error.log, MySQL .err file) for detailed startup failures.
    • Network Connectivity Issues:
      • ping <IP_Address_or_Hostname>: Test basic connectivity to other devices or the internet (e.g., ping google.com or ping 8.8.8.8).
      • tracert <IP_Address_or_Hostname>: Trace the route to a destination to identify where a connection breaks down.
      • netstat -ano: Show active connections and listening ports to confirm your services are listening on the expected ports. Check the PID to identify the process.
      • Check Windows Defender Firewall: Ensure inbound rules are configured to allow traffic to your server's ports (e.g., 80, 443). Also, check if any third-party firewalls are interfering.
      • Check Network Adapter Settings: Verify IP address, subnet mask, gateway, and DNS settings.
